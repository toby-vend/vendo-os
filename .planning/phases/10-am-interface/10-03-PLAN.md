**Add query helpers to `web/lib/queries/drive.ts`:**

1. `listSkillsByChannel(channel: string): Promise<SkillRow[]>` ã `SELECT * FROM skills WHERE channel = ? ORDER BY skill_type ASC, title ASC`. Used for the default browse view (no search query).

2. `getSkillById(id: number): Promise<SkillRow | null>` ã `SELECT * FROM skills WHERE id = ? LIMIT 1`. Used for the skill detail page. Return `result[0] ?? null`.

3. `listSkillChannels(): Promise<string[]>` ã `SELECT DISTINCT channel FROM skills ORDER BY channel ASC`. Used to build the channel tabs dynamically (not hardcoded ã if a new channel is added, the tab appears automatically).

**Create `web/routes/skills-browser.ts`** ã `FastifyPluginAsync` exported as `skillsBrowserRoutes`.

Routes:

1. **GET /** ã Skills browser main page.
   - Parse `channel` from query (default: first available channel, or 'paid_social').
   - Parse `q` (search query) from query.
   - Call `listSkillChannels()` to get available channel tabs.
   - If `q` is non-empty: call `searchSkills(q, channel, 50)` ã returns `SkillSearchResponse`.
   - If `q` is empty: call `listSkillsByChannel(channel)` ã returns all skills for that channel.
   - Group results by `skill_type` for display: `Record<string, SkillRow[]>`.
   - If `request.headers['hx-request']`, render `skills/skill-results` partial only.
   - Otherwise render `skills/browser` full page with `{ channels, activeChannel: channel, query: q, grouped, gap }`.

2. **GET /search** ã HTMX search partial endpoint.
   - Same logic as GET / but always returns `skills/skill-results` partial.
   - Parse `channel` and `q` from query.

3. **GET /:id** ã Skill detail page.
   - Parse `id`, validate.
   - Call `getSkillById(id)`.
   - If not found, 404.
   - If `request.headers['hx-request']`, render `skills/skill-detail` partial (for HTMX modal/panel swap).
   - Otherwise render `skills/skill-detail` as full page with layout.

**Register in `web/server.ts`:**
- Import `skillsBrowserRoutes` from `./routes/skills-browser.js`.
- Register: `app.register(skillsBrowserRoutes, { prefix: '/skills' })`.

**Create `web/views/skills/browser.eta`** ã Skills browser page.
- Layout: `<% layout('/layouts/base') %>`
- Header: "Skills Library" with skill count.
- **Channel tabs**: horizontal tab bar. Each tab is a link: `<a href="/skills?channel=${ch}" class="${ch === it.activeChannel ? 'active' : ''}">`. Tab labels: capitalise and space the channel slug (paid_social -> Paid Social). Use `hx-get="/skills/search?channel=${ch}" hx-target="#skills-results" hx-push-url="true"` for HTMX tab switching without full reload.
- **Search bar**: `<input type="search" name="q" value="${it.query}" placeholder="Search skills..." hx-get="/skills/search" hx-trigger="keyup changed delay:300ms" hx-target="#skills-results" hx-include="[name='channel']">`. Include a hidden input for the active channel.
- **Results container**: `<div id="skills-results"><%~ include('skills/skill-results', it) %></div>`.

**Create `web/views/skills/skill-results.eta`** ã HTMX partial for search results.
- If `it.gap` is true and `it.query` is non-empty: show "No matching skills found for '{query}' in {channel}."
- For each `skill_type` group in `it.grouped`:
  - `<h3>` with skill_type label (capitalised, spaces replacing underscores).
  - Grid or list of skill cards. Each card:
    - Title (linked to `/skills/${skill.id}`).
    - `<span class="badge">${skill.skill_type}</span>`.
    - Last updated: `skill.drive_modified_at` formatted as date.
    - Compact layout ã scannable, not verbose.

**Create `web/views/skills/skill-detail.eta`** ã Full SOP content view.
- Layout: `<% layout('/layouts/base') %>` (when accessed directly).
- Back link to `/skills?channel=${it.skill.channel}`.
- Title: `it.skill.title`.
- Metadata: channel badge, skill_type badge, last updated date, content hash (truncated).
- **Full content**: `<div class="skill-content" style="white-space: pre-wrap; line-height: 1.6;"><%- it.skill.content %></div>`. Use `<%-` (unescaped) only if content is plain text. If content could contain HTML, use `<%=` (escaped). Content from Drive extraction is plain text, so `<%-` wrapped in `<pre>` or `white-space: pre-wrap` is appropriate.
