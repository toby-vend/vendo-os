  <action>
Extend the existing `public/sw.js` by adding Phase 13 code AFTER the existing Phase 12 static asset route. Keep the existing Workbox CDN import, skipWaiting, clientsClaim, and CacheFirst static asset route unchanged.

**CRITICAL: Route registration order matters.** The Phase 12 CacheFirst static asset route MUST remain first. Register the Phase 13 routes after it.

**Add the following in order:**

**1. Precache fallback files at install time (OFFL-01 extension):**
Add an `install` event listener that opens a `vendo-precache-v1` cache and adds `/offline.html` and `/offline-partial.html`. Use `event.waitUntil()` to ensure install does not complete until files are cached. This supplements the existing `skipWaiting()` — do not remove it.

**2. Full-page navigation route (OFFL-02):**
Register a Workbox route for full-page navigations using `workbox.routing.registerRoute`:
- Matcher: `request.mode === 'navigate'` AND `request.headers.get('HX-Request')` is NOT `'true'`
- Strategy: `workbox.strategies.NetworkFirst` with:
  - `cacheName: 'vendo-pages-v1'`
  - `ExpirationPlugin`: maxEntries 50, maxAgeSeconds 7 days
- Handler: Use an async handler function that tries `pageStrategy.handle({ request })` and on catch returns `caches.match('/offline.html')` from the precache
- The try/catch is critical — `NetworkFirst.handle()` throws when both network and cache miss

**3. HTMX partial route (OFFL-03):**
Register a Workbox route for HTMX partial requests:
- Matcher: `request.headers.get('HX-Request') === 'true'` (GET requests only — POST requests are excluded because Workbox only handles GET by default, but add an explicit `request.method === 'GET'` check for clarity)
- Strategy: `workbox.strategies.NetworkFirst` with:
  - `cacheName: 'vendo-partials-v1'`
  - `ExpirationPlugin`: maxEntries 100, maxAgeSeconds 24 hours (partials contain live data, shorter TTL)
- Handler: Try `partialStrategy.handle({ request })`, on catch return `caches.match('/offline-partial.html')`

**Anti-patterns to avoid:**
- Do NOT use `workbox.precaching.precacheAndRoute` — requires build tooling. Use manual `caches.open().addAll()` in the install event.
- Do NOT serve `offline.html` for HTMX partial requests — this injects a full HTML document into a swap target and corrupts the page. The `HX-Request` check MUST happen before the fallback decision.
- Do NOT cache POST requests.
- Do NOT register the Phase 13 routes before the Phase 12 static asset route.
