  <files>web/routes/task-runs.ts</files>
  <action>
Create `web/routes/task-runs.ts` as a FastifyPluginAsync (same pattern as all other route files):

```typescript
import type { FastifyPluginAsync } from 'fastify';
```

Import `createTaskRun`, `getTaskRun`, `listTaskRuns` from `../lib/queries/task-runs.js`.
Import `assembleContext` from `../lib/task-matcher.js`.

Define `VALID_CHANNELS = ['paid_social', 'seo', 'paid_ads'] as const`.

Export `taskRunRoutes: FastifyPluginAsync`:

**POST `/runs`** (becomes POST /api/tasks/runs after prefix):
1. Extract `clientId` (number), `channel` (string), `taskType` (string) from `request.body`
2. Validate: clientId must be a positive integer, channel must be in VALID_CHANNELS, taskType must be a non-empty string. Return 400 `{ error: 'invalid_input', detail: '...' }` on failure.
3. Get user email from `(request as any).user?.email ?? 'unknown'` (session auth is enforced by the onRequest hook — task-runs is not exempted)
4. `const taskRunId = await createTaskRun({ clientId, channel, taskType, createdBy })`
5. `reply.code(202).send({ id: taskRunId, status: 'queued' })` — respond BEFORE context assembly
6. Fire-and-forget: `assembleContext(taskRunId, clientId, channel, taskType).catch(err => request.log.error({ taskRunId, err }, 'context assembly failed'))`

**GET `/runs/:id`**:
1. Parse `id` from `request.params` as integer
2. `const run = await getTaskRun(id)`
3. If null, return 404 `{ error: 'not_found' }`
4. Return 200 with the TaskRunRow

**GET `/runs`**:
1. Parse optional query params: `status`, `clientId`
2. `const runs = await listTaskRuns({ status, clientId })`
3. Return 200 with `{ runs }`

Do NOT add `/api/tasks/runs` to the session auth exemption list — task submission requires an authenticated session.
