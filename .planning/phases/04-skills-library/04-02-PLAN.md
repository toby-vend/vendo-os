  <name>Task 1: Wire FTS5 sync into skill mutation functions</name>
  <files>web/lib/queries/drive.ts</files>
  <action>
    Modify three existing functions in drive.ts to call FTS5 sync helpers inline:

    1. **updateSkillContent()** — After the INSERT/ON CONFLICT upsert, fetch the rowid of the upserted row (`SELECT rowid FROM skills WHERE drive_file_id = ?`), then call `syncSkillFts(rowid, data.title, data.content)`. The rowid fetch is needed because SQLite UPSERT does not return the rowid. Use `scalar<number>()` for the rowid query.

    2. **updateSkillMetadata()** — After the UPDATE, if the title changed the FTS5 title column needs updating. Fetch the current row first (`SELECT rowid, title, content FROM skills WHERE drive_file_id = ?`), then call `syncSkillFts(rowid, data.title, existingContent)`. This re-indexes with the new title but existing content.

    3. **deleteSkill()** — BEFORE the DELETE FROM skills, fetch the row (`SELECT rowid, title, content FROM skills WHERE drive_file_id = ?`), then call `deleteSkillFts(rowid, oldTitle, oldContent)`. The FTS5 delete must happen BEFORE the skills row is deleted because skills_fts is a content-sync table that reads from skills during delete verification.

    Keep changes minimal — do not alter the existing SQL statements, only add the FTS5 sync calls around them.
