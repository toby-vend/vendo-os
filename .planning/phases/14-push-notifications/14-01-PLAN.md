  <action>
1. Create `scripts/generate-vapid-keys.ts` — a one-time utility that imports `web-push`, calls `generateVAPIDKeys()`, and prints the public and private keys to stdout in a format ready to copy into Vercel env vars. Include instructions in the console output: "Set these as VAPID_PUBLIC_KEY and VAPID_PRIVATE_KEY in Vercel environment variables."

2. In `web/lib/queries/auth.ts`, add a new CREATE TABLE statement inside `initSchema()` (after existing tables):
```sql
CREATE TABLE IF NOT EXISTS push_subscriptions (
  id INTEGER PRIMARY KEY,
  user_id TEXT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  endpoint TEXT NOT NULL UNIQUE,
  p256dh TEXT NOT NULL,
  auth TEXT NOT NULL,
  created_at TEXT NOT NULL
);
CREATE INDEX IF NOT EXISTS idx_push_subs_user ON push_subscriptions(user_id);
```
UNIQUE constraint on `endpoint` (not `user_id`) — one row per device, multiple per user. This is a locked decision.

3. Create `web/lib/queries/push-subscriptions.ts` with four exported functions:
- `upsertPushSubscription({ userId, endpoint, p256dh, auth })` — INSERT OR REPLACE keyed on endpoint UNIQUE constraint. Sets `created_at` to current ISO timestamp.
- `deleteSubscriptionByEndpoint(endpoint: string)` — DELETE WHERE endpoint = ?.
- `getSubscriptionsByUserId(userId: string)` — SELECT all rows for user, returns array of `{ endpoint, p256dh, auth }`.
- `countSubscriptionsByUserId(userId: string)` — SELECT COUNT(*) for user, returns number.

Use the same `db` import pattern as other query modules (`import { db } from './db.js'` or however the existing modules import it — check `auth.ts` for the exact pattern).

<task type="auto">
  <name>Task 2: Push sender module, API routes, and server registration</name>
  <files>web/lib/push-sender.ts, web/routes/push.ts, web/server.ts</files>
  <action>
1. Install dependencies: `npm install web-push@3.6.7 && npm install --save-dev @types/web-push`

2. Create `web/lib/push-sender.ts`:
- Export `initVapid()` — calls `webpush.setVapidDetails('mailto:admin@vendoagency.com.au', process.env.VAPID_PUBLIC_KEY!, process.env.VAPID_PRIVATE_KEY!)`. Guard with an early return + console.warn if either env var is missing (do not crash the server if VAPID keys are not yet configured).
- Export `sendPushToUser(userId: string, payload: { title: string; body: string; url: string })`:
  - Calls `getSubscriptionsByUserId(userId)` to get all device subscriptions.
  - Uses `Promise.allSettled` to send to all devices (not `Promise.all` — one failure must not abort others).
  - For each subscription, calls `webpush.sendNotification(sub, JSON.stringify(payload), { TTL: 3600 })`.
  - In the catch block: if `err.statusCode === 410 || err.statusCode === 404`, call `deleteSubscriptionByEndpoint(sub.endpoint)` (PUSH-06 dead subscription pruning — on-send, not scheduled).
  - Log other errors with `console.error('[push] sendNotification failed', ...)` but do not throw.
- Import `getUserByEmail` from `./queries/auth.js` and export `sendPushToUserByEmail(email: string, payload)` — resolves email to userId via `getUserByEmail`, then calls `sendPushToUser`. This is needed because `task_runs.created_by` stores email, not user_id.

3. Create `web/routes/push.ts`:
- Export `pushRoutes` as a `FastifyPluginAsync`.
- `POST /subscribe` — requires authenticated user (`(request as any).user`). Reads `{ endpoint, keys: { p256dh, auth } }` from JSON body. Calls `upsertPushSubscription({ userId: user.id, endpoint, p256dh, auth })`. Returns 201 `{ ok: true }`.
- `DELETE /subscribe` — requires authenticated user. Reads `{ endpoint }` from JSON body. Calls `deleteSubscriptionByEndpoint(endpoint)`. Returns 200 `{ ok: true }`.
- `POST /test` — requires authenticated user. Calls `sendPushToUser(user.id, { title: 'VendoOS', body: 'Test notification', url: '/settings' })`. Returns 200 `{ ok: true }`.
- `GET /vapid-public-key` — returns `{ key: process.env.VAPID_PUBLIC_KEY || '' }`. This gives the client the VAPID public key without server-rendering it into templates.
- `GET /subscription-count` — requires authenticated user. Returns `{ count: await countSubscriptionsByUserId(user.id) }`.

Note: CSRF is already skipped for `/api/` routes (confirmed in server.ts line 276), so no CSRF workaround needed.

4. In `web/server.ts`:
- Add import: `import { pushRoutes } from './routes/push.js';`
- Add registration: `app.register(pushRoutes, { prefix: '/api/push' });` alongside the other API route registrations.
- Add import: `import { initVapid } from './lib/push-sender.js';`
- Call `initVapid()` near the top of the file (after dotenv config, before route registration). This sets VAPID details once at startup.
