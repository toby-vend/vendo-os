1. Create `web/lib/drive-sync.ts` ŌĆö the Google Drive Changes API integration module. This handles channel lifecycle, not webhook receipt.

   Import `getGoogleAccessToken` from `./google-tokens.js` and drive query functions from `./queries.js`. Import `crypto` from Node built-in. Import `getUserOAuthToken` from `./queries.js` for scope checking.

   Export three functions:

   **`registerWatchChannel(userId: string, webhookUrl: string)`** ŌĆö Complete registration sequence:
   a. Call `getGoogleAccessToken(userId)` ŌĆö throw if null ("Admin has not connected Google account")
   b. Check stored scopes via `getUserOAuthToken(userId, 'google')` ŌĆö if `scopes` does not include `drive.readonly`, throw clear error: "Admin must reconnect Google account to grant Drive access"
   c. `GET https://www.googleapis.com/drive/v3/changes/startPageToken` to get `startPageToken`
   d. Generate `channelId = crypto.randomUUID()`
   e. `POST https://www.googleapis.com/drive/v3/changes/watch?pageToken=${startPageToken}` with body: `{ id: channelId, type: 'web_hook', address: webhookUrl, token: process.env.DRIVE_WEBHOOK_SECRET }`
   f. Parse response for `resourceId` and `expiration` (string, parse to int)
   g. Persist to DB via `upsertDriveWatchChannel({ channelId, resourceId, expiration: parseInt(expiration), pageToken: startPageToken, userId })`
   h. Return `{ channelId, resourceId, expiration, pageToken: startPageToken }`

   **`stopWatchChannel(channelId: string, resourceId: string, userId: string)`** ŌĆö Best-effort stop:
   a. Get access token via `getGoogleAccessToken(userId)`
   b. `POST https://www.googleapis.com/drive/v3/channels/stop` with body `{ id: channelId, resourceId }` ŌĆö ignore errors (channel may already be expired)
   c. Delete from DB via `deleteDriveWatchChannel(channelId)`

   **`renewChannel(channel: DriveWatchChannelRow)`** ŌĆö Renew an expiring channel:
   a. Determine userId from `channel.user_id` or fall back to `process.env.DRIVE_ADMIN_USER_ID`
   b. Stop old channel (best-effort) via `stopWatchChannel`
   c. Determine webhookUrl from `process.env.DRIVE_WEBHOOK_URL` or construct from `process.env.VERCEL_PROJECT_PRODUCTION_URL` as `https://${url}/api/drive/webhook`
   d. Register new channel via `registerWatchChannel(userId, webhookUrl)`
   e. Return the new channel data

2. Create `web/routes/drive-webhook.ts` ŌĆö Fastify plugin for `POST /api/drive/webhook`.

   Export `driveWebhookRoutes: FastifyPluginAsync`. Register a POST handler:
   a. Extract headers: `x-goog-channel-id`, `x-goog-resource-state`, `x-goog-channel-token`, `x-goog-resource-id`
   b. Validate `x-goog-channel-token` matches `process.env.DRIVE_WEBHOOK_SECRET` ŌĆö return 403 if not
   c. If `resourceState === 'sync'`, return 200 immediately (initial registration ack)
   d. Look up channel via `getDriveWatchChannel(channelId)` ŌĆö return 404 if not found
   e. Write queue row via `insertDriveSyncQueueItem({ channelId, resourceState })`
   f. Return 200

   No authentication decorator (Google's webhook has no session). The `DRIVE_WEBHOOK_SECRET` token validation is the auth mechanism.

3. Register the webhook route in `web/server.ts`. Find where other route plugins are registered (e.g. `driveRoutes`). Add:

   ```typescript
   app.register(driveWebhookRoutes, { prefix: '/api/drive' });
   ```
   This mounts the POST handler at `/api/drive/webhook`.

   IMPORTANT: The webhook route must NOT be behind the session/auth middleware that protects other routes. Google sends unauthenticated POST requests. Check how the existing route registration works ŌĆö if all routes are wrapped in an auth decorator, the webhook route must be registered outside that wrapper, or the handler must skip session checking. Look at how `web/server.ts` structures its middleware and register the webhook route accordingly.
