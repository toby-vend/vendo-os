    **Query module (brand.ts):**

    Create `web/lib/queries/brand.ts` importing from `./base.js`. Export:

    - Types: `BrandHubRow`, `BrandClientRow`, `BrandSearchResult`
    - `upsertBrandFromDrive(data)` — INSERT ON CONFLICT(drive_file_id) DO UPDATE. Read old row first for FTS5 sync. Call syncBrandFts after upsert. Accepts: driveFileId, title, content, contentHash, clientId (number), clientName, clientSlug, driveModifiedAt.
    - `getBrandContext(clientSlug)` — SELECT * FROM brand_hub WHERE client_slug = ? ORDER BY title ASC. BRND-04: always requires clientSlug parameter.
    - `listBrandClients()` — SELECT client_name, client_slug, client_id, COUNT(*) as file_count FROM brand_hub GROUP BY client_id ORDER BY client_name ASC. Returns names only, never content.
    - `searchBrandContent(query, clientSlug?)` — FTS5 search via brand_hub_fts JOIN brand_hub. Sanitise query (strip quotes, split words, append *). When clientSlug provided, add WHERE b.client_slug = ? (BRND-04). ORDER BY bm25(brand_hub_fts) ASC (negative values = most relevant first). LIMIT 10.
    - `getBrandFile(driveFileId)` — SELECT * FROM brand_hub WHERE drive_file_id = ? LIMIT 1. Returns single row or null.
    - `deleteBrandFile(driveFileId)` — Read old row for FTS5 delete, then DELETE FROM brand_hub WHERE drive_file_id = ?.
    - `syncBrandFts(rowid, oldClientName, oldContent, newClientName, newContent)` — Mirror syncSkillFts exactly: DELETE old tokens then INSERT new. Uses brand_hub_fts(brand_hub_fts, rowid, client_name, content) VALUES('delete', ?, ?, ?) then INSERT.
    - `deleteBrandFts(rowid, clientName, content)` — Delete-only for use during deleteBrandFile.
