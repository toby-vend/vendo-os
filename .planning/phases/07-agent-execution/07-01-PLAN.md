  <action>
    Create `web/lib/task-types/` directory with 4 files.

    **index.ts** — Export a `TaskTypeConfig` interface:
    ```typescript
    export interface TaskTypeConfig {
      schema: Record<string, unknown>;
      buildSystemPrompt: (sopContent: string) => string;
      buildUserMessage: (taskType: string, brandContent: string, clientName?: string) => string;
    }
    ```
    Export `loadTaskTypeConfig(channel: string, taskType: string): TaskTypeConfig` that looks up `${channel}:${taskType}` in a static registry map. Throw if not found.

    **ad_copy.ts** — Paid social / Meta ad copy:
    - Schema: `{ variants: array of 3-5 objects with primary_text, headline, description, call_to_action (all strings, all required), sources: array of {id: number, title: string} }`. All required, additionalProperties: false throughout.
    - buildSystemPrompt: Role = expert paid social copywriter for a digital marketing agency. Instruct to produce Meta ad copy variants grounded in SOPs. Include character guidance in instructions (not just maxLength — per research pitfall 6). Concatenate SOP content under "## SOPs" heading.
    - buildUserMessage: "Generate ad_copy variants for the following brief:" + brand context section if provided.

    **content_brief.ts** — SEO content brief:
    - Schema: `{ meta_title: string, meta_description: string, content_brief: { headings: array of strings, key_points: array of strings, word_count_target: number }, sources: array }`. All required, additionalProperties: false.
    - buildSystemPrompt: Role = expert SEO content strategist. Include char limit guidance (meta_title 60 chars, meta_description 155 chars) in instructions.
    - buildUserMessage: same pattern as ad_copy.

    **rsa_copy.ts** — Paid ads / Google Ads RSA:
    - Schema: `{ headlines: array of strings (minItems 3, maxItems 15), descriptions: array of strings (minItems 2, maxItems 4), sitelink_extensions: array of {title: string, description: string} or empty array, sources: array }`. headlines and descriptions required, sitelink_extensions optional (default empty array). additionalProperties: false.
    - buildSystemPrompt: Role = expert Google Ads copywriter. Include strict char limit guidance (headlines 30 chars each, descriptions 90 chars each) prominently in instructions. Per research pitfall 6: be explicit about counting.
    - buildUserMessage: same pattern.

    - Test loadTaskTypeConfig throws for unknown combo
    - Test each schema has required 'sources' array
    - Test each schema has additionalProperties: false at top level
    - Test buildSystemPrompt includes SOP content string
    - Test buildUserMessage includes brand context with client name heading
    - Test buildUserMessage with empty brand content omits brand section

    Use `node:test` with `describe/it` pattern. No mocks needed — pure functions.
