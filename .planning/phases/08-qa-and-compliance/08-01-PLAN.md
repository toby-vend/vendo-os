Create the deterministic AHPRA dental advertising compliance checker module with the full hardcoded rule set compiled during research. Purpose: AHPRA compliance is a regulatory requirement for all dental advertising content. This module provides the rule-based second pass that runs on every draft before it reaches an AM.

    **Write tests first (RED):**

    **Implement (GREEN):**

    Create `web/lib/ahpra-rules.ts` with:

    1. **Types:** Export `AHPRARule` interface `{ id: string; category: string; description: string; severity: 'HIGH' | 'MEDIUM' | 'LOW'; patterns: RegExp[] }` and `AHPRAViolation` interface `{ rule: string; violation: string; severity: 'HIGH' | 'MEDIUM' | 'LOW' }`.

    2. **Rules array:** Export `AHPRA_RULES: AHPRARule[]` containing ALL rules from the research document (08-RESEARCH.md "AHPRA Dental Advertising Rules" section). The full list:
       - AHPRA-T1: Clinical outcome testimonials (HIGH) — patterns for patient outcome quotes
       - AHPRA-T2: Influencer endorsements (HIGH) — patterns for influencer/ambassador promotion
       - AHPRA-O1: Guaranteed results (HIGH) — "guaranteed", "100% success", "permanent solution", "will fix", "you will achieve", "certain to", "definitely"
       - AHPRA-O2: Unrealistic expectations (MEDIUM) — "pain-free", "instant results", "risk-free", "safe" (unqualified), "miracle", "cure", "permanent" (cosmetic), "happier you", "restore self-esteem", "confidence boost", "transform your life"
       - AHPRA-O3: Fear-based/urgency (MEDIUM) — "book now or" + negative, "before it's too late", "don't risk", "limited time"/"act now" for clinical
       - AHPRA-C1: Superlatives (MEDIUM) — "best", "leading", "most trusted", "world-class", "world renowned", "number one", "top", "premier", "most experienced"
       - AHPRA-C2: Comparative without proof (MEDIUM) — "better than", "unlike other", "the only practice that"
       - AHPRA-Q1: Specialist title misuse (HIGH) — "specialist", "specialises in", "specialty practice", "expert", "expert in" (but NOT "special interest in" — this is explicitly allowed)
       - AHPRA-Q2: Protected title misuse (HIGH) — basic patterns
       - AHPRA-V1: Before-and-after claims (MEDIUM) — copy describing AI-generated comparisons or specific visual outcomes
       - AHPRA-V2: Minors in cosmetic content (HIGH) — targeting under-18s
       - AHPRA-I1: Undisclosed inducements (MEDIUM) — "free" without terms, "limited offer" without terms
       - AHPRA-I2: Unnecessary treatment offers (MEDIUM) — bundle pricing patterns
       - AHPRA-E1: Unsubstantiated claims (MEDIUM) — "proven to", "clinically proven", statistics without source
       - AHPRA-F1: False/misleading (HIGH) — general false claims patterns

    3. **Checker function:** Export `checkAHPRACompliance(draftText: string): AHPRAViolation[]`. For each rule, test each pattern against the full draftText (case-insensitive matching via `RegExp.exec` or `.match`). On match, extract the matched text as the `violation` field. Return all violations found. The `draftText` is the full JSON output serialised as a string — pattern matching runs across all text fields without field-by-field parsing.

    **Critical edge case for AHPRA-Q1:** The phrase "special interest in" is explicitly ALLOWED by AHPRA. The regex for "specialises in" must NOT match "special interest in". Use word boundary `\b` or negative lookbehind to prevent false positives.

    **RegExp compilation:** All patterns in `AHPRA_RULES` are pre-compiled at module load time (they are `RegExp` literals in the array definition). Do NOT create new RegExp objects inside `checkAHPRACompliance`.

    Run tests — all should pass.
